<objective>
Transform the extension from a fire-and-forget step executor into an autonomous agent loop. After each executed action, the extension re-captures the page (screenshot + DOM), sends the updated context plus action history to Nova 2 Lite, and Nova decides whether to issue the next step or signal "done." This repeats until the task is complete or a safety limit of 10 iterations is reached.

Purpose: This is the core NOVA-04 requirement -- the difference between "execute a fixed plan" and "adapt to what actually happened on the page." Without this, multi-step tasks that change the page layout (e.g., clicking a product, then adding to cart on the detail page) cannot be completed.

Output: Working agent loop across backend (continue endpoint + reasoning) and frontend (loop orchestration + bubble progress).
</objective>

3. Replace the `executeSteps()` function with a new `runAgentLoop()` function:

```typescript
async function runAgentLoop(
  tabId: number,
  originalCommand: string,
  initialActions: TaskResponse['actions']
): Promise<void> {
```

This function implements the observe-act loop:

a. If `initialActions` is empty or undefined, send `bubble-state: done` and return.

b. Transition bubble to `executing` state.

c. Initialize: `const actionHistory: ActionHistoryEntry[] = [];` and `let currentActions = initialActions;` and `let iteration = 0;`

d. **Outer loop** (`while (iteration < MAX_AGENT_ITERATIONS)`):
   - Increment iteration.
   - **Inner loop**: Execute each action in `currentActions` one at a time:
     - Send `bubble-step` with `stepName: step.description`, `stepIndex` as the action history length + 1, `totalSteps` as `0` (unknown total in agent mode -- use 0 to signal "iterating").
     - Send `execute-action` to content script via `chrome.tabs.sendMessage()` (same pattern as old executeSteps).
     - On content script unreachable: send `pipeline-error`, return.
     - On `result.ok === false`: send `pipeline-error`, return.
     - Send `bubble-step` with `stepName: result.summary` (post-action feedback, same as Phase 9 pattern).
     - Push `{ description: step.description, result: result.summary }` to `actionHistory`.
     - Brief 500ms pause after action to let the page settle (DOM updates, animations, network requests).

   - After executing all actions in current batch: **Re-observe the page**.
     - Wait 800ms for page to fully settle after last action (navigation, AJAX, animations).
     - Re-capture screenshot: `const newScreenshot = await captureScreenshot(tabId);`
     - Re-scrape DOM: send `scrape-dom` message to content script, extract snapshot (same pattern as runPipeline). If scrape fails, use empty object `{}`.
     - Show bubble state: send `bubble-state: understanding` with label `"Re-evaluating... (${iteration}/${MAX_AGENT_ITERATIONS})"`.

   - **Ask Nova what to do next**: call `sendTaskContinue(originalCommand, actionHistory, newScreenshot, domSnapshot)`.
     - On error: send `pipeline-error`, return.

   - **Evaluate Nova's response**:
     - If `result.type === 'done'`: send `bubble-state: done`, return. (Task complete!)
     - If `result.type === 'answer'`: This means Nova wants to communicate something. Show the answer in the bubble (same pattern as runPipeline answer handling: `bubble-state: answering`, `bubble-answer-chunk`, `bubble-answer-done`, `tts-summary`), then return.
     - If `result.type === 'steps'`: set `currentActions = result.actions` and continue the outer loop.
     - If `result.actions` is empty/undefined when type is 'steps': treat as done.

e. After the outer loop exits (max iterations reached): send `bubble-step` with `stepName: "Reached maximum iterations (${MAX_AGENT_ITERATIONS})"`, then `bubble-state: done`.

4. **Update `runPipeline()`**: In the `else if (taskResult.type === 'steps')` branch, replace `await executeSteps(tabId, taskResult.actions);` with `await runAgentLoop(tabId, transcript, taskResult.actions);`.

5. **Update `runFollowUp()`**: Same change in its steps branch -- replace `await executeSteps(tabId, taskResult.actions);` with `await runAgentLoop(tabId, text, taskResult.actions);`.

6. **Remove the old `executeSteps()` function entirely** -- it is fully replaced by `runAgentLoop()`.

IMPORTANT: Preserve all existing conversation history logic in runPipeline and runFollowUp -- the history push happens BEFORE the agent loop call, which is the current pattern and should stay.
