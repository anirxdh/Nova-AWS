  <action>
**In `src/shared/types.ts`:** Add two new entries to the `MessageType` union:

```typescript
| { action: 'execute-action'; actionType: string; selector?: string; value?: string; url?: string; direction?: string; description: string }
| { action: 'action-result'; ok: boolean; summary: string; error?: string }
```

**Create `src/content/action-executor.ts`:** This module exports `executeAction(request)` which:

1. **Allowlist check** -- Only these action types are allowed: `click`, `type`, `navigate`, `extract`, `scroll`. If the incoming `actionType` is not in this set, return `{ ok: false, summary: '', error: 'Unknown action type: {actionType}' }`.

2. **Selector sanitizer** -- Before any `document.querySelector(selector)` call, validate the selector:
   - Reject if selector contains `javascript:`, `<script`, `on` + event name patterns (e.g., `onclick=`), or backtick characters
   - Use a try/catch around `document.querySelector(selector)` to catch malformed selectors
   - If selector is required but missing or invalid, return error result

3. **Rate limiter** -- Track `lastActionTime` at module scope. If less than 300ms since the last action, wait the remaining time with a setTimeout/Promise before executing. This prevents runaway loops from the agent.

4. **Action implementations:**

   - `click(selector)` -- `document.querySelector(selector)` -> cast to HTMLElement -> `.click()`. If element not found, return error. Summary: `"Clicked '{element.textContent?.trim().slice(0,50) || selector}'"`.

   - `type(selector, value)` -- `document.querySelector(selector)` -> cast to HTMLInputElement -> set `.value = value` -> dispatch `new Event('input', { bubbles: true })` then `new Event('change', { bubbles: true })`. Also `.focus()` the element before setting value. Summary: `"Typed '${value.slice(0,30)}' into ${selector}"`.

   - `navigate(url)` -- Validate url starts with `http://` or `https://` (reject `javascript:`, `data:`, `blob:` schemes). Set `window.location.href = url`. Summary: `"Navigating to ${url}"`. Return ok immediately (page will unload).

   - `extract(selector)` -- `document.querySelector(selector)` -> get `.textContent?.trim()`. Summary: `"Extracted text from ${selector}: '${text.slice(0,100)}'"`. Include the extracted text in the result.

   - `scroll(selector_or_direction)` -- If value is `up`/`down`/`top`/`bottom`, use `window.scrollBy` or `window.scrollTo`. Otherwise treat as selector: `document.querySelector(selector)` -> `.scrollIntoView({ behavior: 'smooth', block: 'center' })`. Summary: `"Scrolled to ${selector_or_direction}"`.

5. **Return type:**
```typescript
export interface ActionRequest {
  actionType: string;
  selector?: string;
  value?: string;
  url?: string;
  direction?: string;
  description: string;
}

export interface ActionResult {
  ok: boolean;
  summary: string;
  error?: string;
  extractedText?: string;
}
```

Each action is wrapped in try/catch. On any DOM error, return `{ ok: false, summary: '', error: error.message }`.

**In `src/content/content-script.ts`:**

1. Add import at top: `import { executeAction } from './action-executor';`

2. In the `chrome.runtime.onMessage.addListener` callback, add a new handler BEFORE the existing `if (message.action === 'scrape-dom')` block (since execute-action also needs async sendResponse):

```typescript
if (message.action === 'execute-action') {
  // Execute the DOM action and return result asynchronously
  executeAction({
    actionType: message.actionType,
    selector: message.selector,
    value: message.value,
    url: message.url,
    direction: message.direction,
    description: message.description,
  }).then((result) => {
    sendResponse(result);
  });
  return true; // keep message channel open for async sendResponse
}
```

This allows the service worker to `chrome.tabs.sendMessage(tabId, { action: 'execute-action', ... })` and await the result.

Do NOT modify any existing message handlers. The execute-action handler must be placed before the scrape-dom handler since both use async sendResponse with `return true`.
