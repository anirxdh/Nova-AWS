  <action>
Create `src/content/cursor-bubble.ts` — the unified cursor-following UI component. This replaces both `Overlay` and `ListeningIndicator`.

**Architecture:**
- Single class `CursorBubble` with a state machine driven by `setState(state: BubbleState)`
- Shadow DOM isolated (closed mode), max z-index (2147483647)
- Host container: `position:fixed; pointer-events:none;` so it never blocks page interaction
- The bubble element inside: `pointer-events:auto` ONLY when the user needs to interact (follow-up input when answering is complete) — for all other states keep `pointer-events:none`
- Inline styles constant (same pattern as current overlay.ts — all CSS in a template literal)
- Mouse tracking: `document.addEventListener('mousemove', ...)` with `{ passive: true }` — active from show() to dismiss()

**Visual design (dark glass, matches existing aesthetic):**
- Background: `rgba(20, 20, 20, 0.85)` with `backdrop-filter: blur(16px)`
- Border: `1px solid rgba(255, 255, 255, 0.1)`
- Border-radius: `14px`
- Font: `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- Text color: `rgba(255, 255, 255, 0.9)`
- Compact size for status states (~180px wide), expanding for answer text (~380px wide, max-height 360px with overflow-y auto)

**State rendering (inside the bubble element):**

1. **listening**: Show animated waveform bars (reuse the 16-bar pattern from ListeningIndicator). The bars are driven by `updateAmplitude(frequencyData: Uint8Array)`. Also show a small "Listening..." label below the bars in `rgba(255, 255, 255, 0.4)` font-size 11px.

2. **transcribing**: Show a pulsing dot (CSS animation) + "Transcribing..." text. Small compact bubble.

3. **understanding**: Same layout as transcribing but text says "Understanding...".

4. **planning**: Same layout but text says "Planning...".

5. **executing**: Show a step indicator: "Step {n}/{total}: {stepName}" — e.g., "Step 1/3: Clicking Add to Cart...". Use `setStep(name: string, index: number, total: number)` method. Show a small progress feel — muted step count, active step name.

6. **answering**: Expand the bubble to answer width (~380px). Render streaming text via `appendChunk(text: string)` method using `renderMarkdown()` from `./markdown`. Auto-scroll as content grows. When answer is complete (triggered by `onAnswerDone()`), optionally show a follow-up input row (only if displayMode is not 'audio-only'). Include the TTS toggle button and context bar (conversation turns indicator + Clear button) from the existing overlay pattern.

7. **error**: Show error text in `rgba(255, 120, 120, 0.9)`. Auto-dismiss after 5 seconds.

8. **done**: Show a check icon or "Done" text. Auto-dismiss after 2 seconds with fade-out animation.

9. **idle**: Bubble is hidden / removed from DOM.

**Methods (public API):**
- `show(cursorX: number, cursorY: number): void` — Create Shadow DOM container, attach to body, start mouse tracking. Initial state should be set via setState immediately after.
- `setState(state: BubbleState, label?: string): void` — Transition bubble to new state. Clears previous state content, renders new state. If state is 'idle', calls dismiss.
- `updateAmplitude(frequencyData: Uint8Array): void` — Update waveform bar heights (only meaningful in 'listening' state).
- `setStep(name: string, index: number, total: number): void` — Update step display (only meaningful in 'executing' state).
- `appendChunk(text: string): void` — Append streaming text (only meaningful in 'answering' state). First call transitions to answering state automatically if not already.
- `onAnswerDone(): void` — Signal that answer streaming is complete. Shows follow-up input if displayMode allows.
- `speakSummary(summary: string): void` — Trigger TTS. In audio-only mode, show waveform and auto-dismiss when TTS finishes.
- `showError(error: string): void` — Convenience: sets state to error with the error message.
- `dismiss(): void` — Fade out and remove from DOM. Stop mouse tracking, clear timers.
- `isVisible(): boolean` — Returns whether bubble is currently shown.
- `hideForScreenshot(): void` — Temporarily hide (display:none) for screenshot capture.
- `showAfterScreenshot(): void` — Re-show after screenshot.
- `setCallbacks(onFollowUp: (text: string) => void, onClear: () => void): void` — Register callbacks for follow-up and clear actions.
- `updateConversationInfo(info: ConversationInfo): void` — Update the context bar turns indicator.
- `prepareForFollowUp(): void` — Move current Q&A into history, clear response area, restart mouse tracking.

**Positioning logic:**
Reuse the edge-detection logic from overlay.ts `positionOverlay()` method — when bubble is near screen edges, flip it so it stays visible. The bubble should appear ~20px below the cursor, centered horizontally.

**Key implementation notes:**
- Import `renderMarkdown` from `./markdown` for answer text rendering
- Import `speak`, `stop as stopTts`, `isTtsEnabled`, `setTtsEnabled`, `isSpeaking` from `./tts` for TTS integration
- Import `getSettings` from `../shared/storage` for displayMode
- Import `BubbleState`, `ConversationInfo`, `DisplayMode` from `../shared/types`
- CSS animations: `@keyframes` for pulsing dot, waveform bars, fade-in/fade-out
- The bubble must re-render cleanly when setState is called — clear inner content and rebuild for the new state
- Preserve chat history across follow-ups (same pattern as current overlay: move Q&A into history section before clearing)
- The host container (`#screensense-bubble-host`) should be a div with `position:fixed;top:0;left:0;width:0;height:0;z-index:2147483647;pointer-events:none;`
- Escape key handler (capture phase) to dismiss the bubble

**Do NOT:**
- Import from overlay.ts or listening-indicator.ts at runtime (those will be removed in Plan 02)
- Use external CSS files (everything must be inlined in the Shadow DOM for isolation)
- Make the bubble interactive during status states — only `pointer-events:auto` on the bubble element when follow-up input is shown
